# ZChat Backend

Бэкенд мессенджера с приватными диалогами, группами, текстовыми и голосовыми
каналами и доставкой событий в реальном времени по WebSocket.

## Стек

- Go + Gin
- PostgreSQL
- Redis
- JWT

## Структура

Код разбит на bounded contexts. Каждый контекст содержит свои сущности,
сервис, репозиторий и HTTP-обработчики.

```
cmd/server/    — точка входа
config/        — конфигурация из переменных окружения

internal/
  apperror/    — общие ошибки
  httpapi/     — общая HTTP-инфраструктура
  platform/    — http server, postgres, redis, logger

  auth/        — пользователи, JWT, сессии
  group/       — группы, каналы, участники, роли
  chat/        — личные чаты, сообщения, статусы прочтения
  voice/       — аудит модерации голосовых каналов
  realtime/    — presence, pub/sub, состояние голосовых
    ws/        — WebSocket-транспорт

  app/         — composition root
```

## Быстрый старт

1. Скопировать шаблон окружения и заполнить секреты:

```bash
cp .env.example .env
```

2. Поднять зависимости и приложение:

```bash
make docker-up
```

3. Проверить, что сервис жив:

```bash
curl http://localhost:8080/health
```

Полезные команды:

```bash
make docker-logs
make docker-down
```

## Миграции

SQL-миграции лежат в `migrations/` и применяются в порядке нумерации.

## Роли в группах

- `owner` — полный контроль группы и её каналов.
- `admin` — управление каналами и обычными участниками.
- `member` — чтение и запись в доступных каналах.

## Realtime

WebSocket доступен по `GET /api/v1/ws` (с `?token=` или заголовком
`Authorization: Bearer ...`). Через него идут подписки на каналы и личные
чаты, события presence и сигналинг для голосовых каналов.
